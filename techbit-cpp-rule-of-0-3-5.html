Rule of 0, 3, 5

Move Symantics

Motive:
  1. Avoid expensive deep copy.
  2. Reuse memory allocated for temporary variables (e.g. rvalue).
Before we begin:
 
let's understand:
Rule of 0:
Better to avoid defining any of above mentioned member functions.
 
Rule of 3:
If a class requires any of following, it is certain that it will require all 3.
  1. copy constructor
  2. copy assignment operator
  3. destructor
If a class does not provide any then default implementation will be provided by the compiler.
 
Rule of 5:
Any class for which move semantics are desired, it will need to define all 5 special functions
Above mentioned 3 + following 2.
  1. 4) move-constructor
  2. 5) move assignment operator
 
copy-and-swap idiom
